Entender a base de dados

  • instant: ndice de registro
  • dteday: data
  • season: estao (1: inverno, 2: primavera, 3: vero, 4: outono)
  • yr: ano (0: 2011, 1: 2012)
  • mnth: ms (1 a 12)
  • hr: hora (0 a 23)
  • holiday: o dia com clima feriado ou no (extrado de [Web Link])
  • weekday: dia da semana
  • workingday: se o dia no for final de semana nem feriado, for 1, caso contrrio ser 0 - equivalente ao dia til.
  • weathersit: tempo
  • 1: limpo, poucas nuvens, parcialmente nublado, parcialmente nublado
  • 2: nvoa + nublado, nvoa + nuvens partidas, nvoa + poucas nuvens, nvoa
  • 3: neve fraca, chuva fraca + trovoada + nuvens dispersas, chuva fraca + nuvens dispersas
  • 4: chuva forte + paletes de gelo + trovoada + nvoa, neve + nevoeiro
  • temp: temperatura normalizada em graus Celsius. Os valores so derivados de (t-t_min) / (t_max-t_min), t_min = -8, t_max = + 39 (apenas na escala horria)
  • atemp: temperatura normal de sensao em graus Celsius. Os valores so derivados de (t-t_min) / (t_max-t_min), t_min = -16, t_max = + 50 (apenas na escala horria)
  • hum: umidade normalizada. Os valores so divididos em 100 (mximo)
  • windspeed: velocidade do vento normalizada. Os valores so divididos em 67 (mximo)
  • casual: nmero de usurios casuais
  • registered: nmero de usurios registrados
  • cnt: contagem do total de bicicletas alugadas, incluindo bicicletas casuais e registradas
